## 本周目标

- 继续推进 Phase 3 自部署同步服务端前置治理。
- 在真实 HTTP handler 和远端上传下载前，补齐 Go server storage 层的签名验签门禁。

## 风险与未完成项

- 当前仍不实现 Docker Compose、Rust 真实 HTTP transport、两客户端端到端同步、Flutter manager 或平台壳。
- 当前未启动长期运行 server 做真实联调，也未编写 Docker Compose 或部署配置。
- Device wrapping record 已保存 wrapped key metadata 与 encrypted wrapped key bytes；真实授权 handler 对外开放时仍必须只返回密文 bytes 和服务端可见 metadata。
- Recovery latest handler 已能返回 metadata 与 encrypted wrapped material bytes，并覆盖限速与内部 `blob_ref` 不外泄；API 外层已补 panic recovery、request id、非持久审计 hook 和 SQLite audit_events 写入；runtime audit logger、本机 runbook 和短生命周期 HTTP smoke 已覆盖请求体脱敏和对象上传下载复验。
- Go recovery record 现在持久化 KDF 参数以支持验签；后续 HTTP DTO 和 Rust 真实 HTTP transport 接线时必须继续保持这些字段只描述加密恢复材料，不包含恢复码明文或 KDF 输出。
