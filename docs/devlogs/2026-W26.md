# 2026-W26 开发日志

本文档记录 2026-W26 周内 RadishLex 的阶段性推进事实、验证证据和后续建议，读者是项目维护者和后续 AI / 人工协作者。本文不承载长期架构设计、接口字段完整参考或历史推演；长期边界应回写到对应专题文档。

## 本周目标

- 从 Phase 0 方案冻结推进到 Phase 1 Rust Core 原型起步。
- 先固化 Rust core 与底层输入引擎之间的稳定边界。

- 在 Phase 1 证据链稳定后，推进 Phase 2 本地学习、候选重排、同步边界草案和 FFI 边界起步验证。

## 关键决策

- Phase 1 起步只定义平台无关核心模型，不接 `librime`、SQLite、Go server、Flutter 或平台 SDK。
- test-only stub engine 只用于生命周期验证，不作为真实中文输入引擎。
- CLI demo 使用合成候选，只验证 core 调用路径，不代表真实输入质量。
- `ime-engine-rime` 默认不 vendor `librime`，默认 CI / workspace 检查不依赖本机 native Rime 环境。
- `native-rime` feature 已完成本机依赖探测、session 生命周期、输入处理、context 读取、commit 读取、首候选提交、非首候选提交、翻页候选提交和候选索引异常路径 smoke。
- `ime-cli rime` 作为真实 adapter 的 CLI 入口，但真实候选质量与候选选择行为仍以本机 `librime` smoke 为准。
- `ime-core` 不暴露底层 engine 私有对象 ID；后续 ranker 与学习只依赖 RadishLex candidate。
- Phase 1 的 Rust core / CLI / Rime adapter 证据链已满足进入 Phase 2 个人化学习设计的条件；下一步不直接进入平台壳。
- Phase 2 的原始选择事件、负反馈详细事件和应用上下文统计按 P1 处理，默认仅本地学习；用户词库、权重摘要和自定义短语按 P2 处理，后续只能加密同步。
- `ime-userdb` 起步实现必须继续保持本地优先和隐私分级：P0 不写入，P1 原始事件默认只本地保存，P2 用户词库和摘要后续再进入加密同步设计。
- `ime-ranker` 不访问 SQLite 或底层 engine，只消费 RadishLex candidate、userdb summary 和 deleted tombstone summary，并输出结构化 explain。
- `ime-cli` 学习命令必须显式传入 `--db`；当前只用于临时 SQLite 数据库和合成词，不隐式读取真实用户输入法目录。
- `ime-cli rime --rank-db` 只读取显式传入的隔离 userdb；重排后的候选索引会映射回原始 engine index 再调用 `commit_candidate`。
- `dict import/export` 只处理用户词条层的 P2 数据，不导出 selection event、negative feedback 明细、上下文统计或 ranker 权重摘要；普通导入不得复活 deleted tombstone。
- `dict import --dry-run` 不写入 userdb 或 import batch；实际导入必须记录 batch，并明确区分新插入、覆盖更新、删除跳过和导入文件内重复跳过。
- `dict inspect` 是无数据库依赖的导入文件兼容检查；同步前置检查只输出计数和分类，不输出用户词明文、P1 事件明文或同步 payload。
- `ime-sync` 当前承载 payload 边界、加密对象外壳、同步域、设备状态、加入请求、授权包、撤销记录和对象版本冲突草案模型；上传下载、Go server 存储、签名、生产恢复码和客户端冲突合并执行器仍属于后续阶段。
- 平台壳进入前必须先让 `ime-ffi` 明确所有权、生命周期、错误语义、字符串编码和释放责任，不能让平台端直接持有 Rust 内部对象、Rime 私有对象或 SQLite handle。
- `ime-ffi` 当前验证 C ABI 生命周期、ABI contract、session owner-thread policy、释放 panic 边界、结构化 snapshot / candidate ABI、normalized key event、engine kind 门禁、Rime session options 默认 unavailable 门禁、`native-rime` feature 下真实 Rime session smoke、native Rime 错误映射、sync preflight 状态摘要、learning status 只读摘要、受控 userdb add / delete / list、dictionary inspect / export / import、import batches 只读查询、平台绑定式 view copy / release 和 demo engine host smoke；真实平台接入前仍需由具体平台 wrapper 复验线程调度、字符串复制和释放规则。
- 真实 Rime 进入 `ime-ffi` 时使用独立 `RadishLexRimeSessionOptions` 与 `radishlex_session_new_rime`，不复用通用 `engine_kind` 字段承载路径和 schema；默认构建下该入口校验参数后返回 `InvalidState`，不得静默退回 demo engine。
- `docs/technical-plan.md` 只作为技术方向入口，不再承载完整字段参考、验证流水或长篇阶段推演；细节统一进入对应专题文档。
