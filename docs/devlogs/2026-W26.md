## 关键决策

- Phase 1 起步只定义平台无关核心模型，不接 `librime`、SQLite、Go server、Flutter 或平台 SDK。
- test-only stub engine 只用于生命周期验证，不作为真实中文输入引擎。
- CLI demo 使用合成候选，只验证 core 调用路径，不代表真实输入质量。
- `ime-engine-rime` 默认不 vendor `librime`，默认 CI / workspace 检查不依赖本机 native Rime 环境。
- `native-rime` feature 已完成本机依赖探测、session 生命周期、输入处理、context 读取、commit 读取、首候选提交、非首候选提交、翻页候选提交和候选索引异常路径 smoke。
- `ime-cli rime` 作为真实 adapter 的 CLI 入口，但真实候选质量与候选选择行为仍以本机 `librime` smoke 为准。
- `ime-core` 不暴露底层 engine 私有对象 ID；后续 ranker 与学习只依赖 RadishLex candidate。
- Phase 1 的 Rust core / CLI / Rime adapter 证据链已满足进入 Phase 2 个人化学习设计的条件；下一步不直接进入平台壳。
- Phase 2 的原始选择事件、负反馈详细事件和应用上下文统计按 P1 处理，默认仅本地学习；用户词库、权重摘要和自定义短语按 P2 处理，后续只能加密同步。

- `cargo check -p radishlex-ime-cli --features native-rime` 在 Homebrew `librime` 1.17.0 环境下通过。
