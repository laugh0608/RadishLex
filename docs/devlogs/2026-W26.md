## 关键决策

- Phase 1 起步只定义平台无关核心模型，不接 `librime`、SQLite、Go server、Flutter 或平台 SDK。
- test-only stub engine 只用于生命周期验证，不作为真实中文输入引擎。
- CLI demo 使用合成候选，只验证 core 调用路径，不代表真实输入质量。
- `ime-engine-rime` 默认不 vendor `librime`，默认 CI / workspace 检查不依赖本机 native Rime 环境。
- `native-rime` feature 已完成本机依赖探测、session 生命周期、输入处理、context 读取、commit 读取和首候选提交 smoke；非首候选和翻页选择后续补测。
- `ime-cli rime` 作为真实 adapter 的 CLI 入口，但真实候选质量与候选选择行为仍以本机 `librime` smoke 为准。
- `ime-core` 不暴露底层 engine 私有对象 ID；后续 ranker 与学习只依赖 RadishLex candidate。

- `cargo test -p radishlex-ime-core` 通过，3 个测试通过。

- `cargo check -p radishlex-ime-engine-rime --features native-rime` 在临时 fake include/lib 探测目录下通过，用于确认 Rust 侧 native feature 类型检查；该命令不链接或运行真实 `librime`。
- `cargo check -p radishlex-ime-cli --features native-rime` 在临时 fake include/lib 探测目录下通过，用于确认 CLI 能在 native feature 下接到 Rime adapter 类型；该命令不链接或运行真实 `librime`。

- `cargo run -p radishlex-ime-cli --features native-rime -- rime --schema luna_pinyin ... luobo` 在隔离 Rime 数据目录下输出 `composition: luo bo`、5 个候选和 `commit: 蘿蔔`。
- `./scripts/check-repo.sh` 通过。
- `ime-engine-rime` 默认测试覆盖配置、错误展示、key 分类和候选转换，不需要本机 `librime`。
